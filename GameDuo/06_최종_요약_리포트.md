# 최종 요약 리포트 — 게임듀오 게임 데이터 분석가 지원 포트폴리오

## 프로젝트 개요

이 프로젝트는 게임듀오의 "게임 데이터 분석가" 채용 공고에 지원하기 위해, 공고에 명시된 요구 역량을 직접 손으로 증명하는 것을 목표로 진행했다. 실존하는 공개 데이터셋을 가져다 쓰는 대신, 게임듀오가 실제로 서비스하는 게임(아처키우기 등)과 같은 장르인 방치형(Idle) RPG를 가정해서 50만 명 규모의 유저 행동 데이터를 처음부터 직접 설계하고 생성했다. 데이터는 로컬이 아니라 실제 GCP BigQuery 프로젝트 위에서 생성·가공했고, 그 과정에서 겪은 실제 문제(파티션 자동 만료 버그)를 원인부터 추적해서 해결한 경험까지 포함되어 있다. 전체 작업 과정과 매 단계의 의사결정 이유는 `00_프로젝트_진행일지.md`에 시간 순으로 기록해뒀다.

## 핵심 성과 요약

시뮬레이션한 유저는 50만 명, 관찰 기간은 365일이며, 이 기간 동안 세션 736만 2,135건, 행동 이벤트 9,202만 1,438건, 결제 6만 2,847건(누적 매출 292만 2,503달러)이 발생했다. 설계한 리텐션 곡선은 D1 47.3%, D7 20.7%, D30 6.3%로 목표치(D1 45%, D7 20%, D30 8%)에 근접하게 검증됐고, 결제 전환율은 4.29%, 상위 0.5%인 웨일 세그먼트가 전체 매출의 69.0%를 차지하는 전형적인 F2P 매출 구조를 재현했다.

채널별로는 오가닉 유입이 D7 리텐션 25.5%로 가장 높았고, 유료 채널 중에서는 Applovin이 CPI 대비 ROAS 1.40으로 가장 효율적이었던 반면 TikTok은 ROAS 0.79로 스펜드 대비 손실 구간에 있었다. A/B 테스트에서는 가챠 확률 부스트 실험이 결제 전환율을 4.42%에서 7.63%로 통계적으로 매우 유의미하게(p<0.000001) 끌어올린 반면, 튜토리얼 개편 실험은 D1 리텐션에 유의미한 차이를 만들지 못했다(p=0.64) — 서로 다른 결론이 나오는 두 실험을 통해 "효과를 찾아내는 것"과 "효과가 없음을 올바르게 판단하는 것" 둘 다 다룰 수 있었다. 이탈 예측 모델(BigQuery ML 로지스틱 회귀)은 ROC-AUC 0.565로 예측력 자체는 높지 않았지만, 피처 가중치를 분석해서 그 원인이 유입 채널의 영향력이 압도적으로 크게 설계된 데이터 구조 때문이라는 것을 정확히 짚어냈고, 위험도 상위/하위 10분위 간 실제 리텐션 차이는 19%p로 실무적 타겟팅에는 활용 가능한 수준이었다.

## 산출물 가이드

`00_프로젝트_진행일지.md`는 프로젝트 전체의 사고 과정을 시간 순으로 담은 핵심 문서로, 각 의사결정의 이유와 겪었던 문제 해결 과정까지 포함되어 있어 이 프로젝트를 가장 깊이 이해할 수 있는 자료다. `01_JD_분석.md`와 `02_데이터_스키마_설계.md`는 기획 단계 산출물이고, `03_AB테스트_분석.md`와 `04_이탈예측_모델.md`는 분석 결과 리포트, `05_자동화_파이프라인.md`는 파이프라인 설계 문서다. `게임듀오_대시보드.html`은 브라우저에서 바로 열리는 인터랙티브 대시보드이며, `dbt_project/`와 `airflow_dags/`는 실행 가능한 코드 산출물이다. 원본 데이터와 요약 테이블은 GCP 프로젝트 `newproject-502815`의 BigQuery 데이터셋 `dungeon_game_analytics`에 실제로 존재한다.

## 회고 — 이 프로젝트에서 배운 것

가장 크게 배운 건 "숫자가 이상하면 반드시 이유가 있다"는 것이었다. 세션 데이터가 예상보다 훨씬 적게 나왔을 때, 처음엔 날짜 설정이나 무작위 함수의 문제로 의심했지만 하나씩 가설을 세우고 배제해나간 끝에 결국 GCP 샌드박스 모드의 숨겨진 파티션 만료 정책이 원인이었다는 걸 찾아냈다. 에러 메시지 하나 없이 데이터가 조용히 줄어드는 상황에서, 작은 샘플로 실험을 쪼개 어느 단계에서 문제가 생기는지 좁혀나가는 접근이 실제 데이터 분석 업무에서도 그대로 쓰일 수 있는 방법이라는 걸 체감했다.

또 하나는, 좋은 분석 결과만큼이나 "안 되는 이유를 설명할 수 있는 것"도 중요하다는 점이다. 이탈 예측 모델의 AUC가 낮게 나왔을 때 거기서 멈추지 않고 피처 가중치를 뜯어봐서 그 원인이 데이터 자체의 구조에 있다는 걸 확인했고, A/B 테스트에서도 하나는 유의미한 효과를, 하나는 유의미하지 않은 결과를 정확히 판별해내는 경험을 의도적으로 둘 다 만들어봤다. 마지막으로, 처음부터 대규모·실제 클라우드 환경으로 진행하기로 한 선택 덕분에 결제 계정 연결, 비용 관리, 샌드박스 제약 같은 실무에서 실제로 마주치는 문제들을 로컬 시뮬레이션이었다면 겪지 않았을 방식으로 직접 부딪혀볼 수 있었다.

## 한계와 향후 개선 방향

이 프로젝트의 데이터는 실제 유저 데이터가 아니라 설계한 규칙(리텐션 곡선, 결제 확률)에 따라 생성한 시뮬레이션이라는 근본적 한계가 있다. 특히 이탈 예측 모델에서 확인했듯, 초반 행동과 장기 리텐션 사이의 관계가 실제 게임보다 단순하게 설계되어 있어 모델의 예측력이 제한적이었다. 실제 데이터로 다시 작업한다면 더 복잡한 인과관계(예: 특정 컨텐츠 소비 패턴과 리텐션의 관계)를 모델에 반영할 수 있을 것이고, 인과추론 기법(DID, PSM 등)을 활용해 마케팅 채널의 순수한 효과를 더 정교하게 분리해내는 작업도 다음 단계로 시도해볼 만하다.
